# Phase 게이트 (Web)

각 Phase의 **진입 조건 / 산출물 / 종료 조건**을 정의. iOS의 Phase 0~5 구조를 Web 스택으로 이식.

## Phase 0 — 요구사항 명세

**진입**: 사용자 요청을 받았다.
**할 일**:
- 한 문장으로 검증 가능한 동작 기술 ("X 하면 Y 된다")
- 영향받는 화면/도메인/SDK API 식별
- iOS 본앱에 같은 기능 있는지 확인
**산출물**: 테스트 이름 후보 1~3개
**종료**: "이 동작을 증명할 테스트 이름"이 한 문장으로 나옴

## Phase 1 — RED

**진입**: Phase 0 완료
**할 일**:
- 가장 외곽(사용자 시나리오) 테스트부터 작성
- hook → API 순으로 안쪽 테스트 추가
- 프로덕션 코드 절대 수정 금지
**산출물**: `*.test.ts(x)` 파일
**종료**:
- `npm test`로 새 테스트가 **실제 실패**
- 실패 사유가 우리가 의도한 사유와 일치 (단순 오타/import 누락 아님)

## Phase 2 — GREEN

**진입**: Phase 1 완료, 빨간 테스트 확보
**할 일**:
- 그 테스트만 통과시키는 최소 구현
- `any`/`as` 우회 금지
- 의존성 주입 지점은 모킹 가능한 형태(함수/모듈 단위)로
**산출물**: 새 컴포넌트/훅/API 함수
**종료**: 새 테스트 GREEN, 기존 테스트 GREEN

## Phase 3 — REFACTOR

**진입**: Phase 2 완료
**할 일**:
- 중복 제거, 이름 다듬기, 추출
- 한 번에 하나의 변경
- 매 변경 후 `npm test` 통과 확인
**산출물**: 깔끔해진 같은 동작
**종료**: 전체 GREEN, 더 다듬을 게 없음

## Phase 4 — GREEN' (재검증)

**진입**: Phase 3 완료
**할 일**:
```bash
npm run lint
npm run build
npm test
```
**산출물**: 통과 로그
**종료**: 셋 다 0 exit code

## Phase 5 — 리뷰 & 커밋

**진입**: Phase 4 완료
**할 일**:
- `git diff` 자가 점검 (의도 외 변경 없음)
- [git/push-check.md](../git/push-check.md) 6항목 통과
- 커밋 메시지 작성 ([git/commit.md](../git/commit.md))
**산출물**: 커밋 (또는 PR)
**종료**: 푸시 가능 상태

## 게이트 위반 시

위 Phase 중 하나라도 산출물 없이 다음으로 넘어갔다면 **즉시 되돌아간다**.
"나중에 쓰면 되지" 금지. 한 번 깨진 게이트는 누적된다.
