# 워크플로우: 리팩토링

## 정의

**동작 변경 없이 구조만 바꾸는 작업.**
동작이 바뀌면 그건 리팩이 아니라 기능/버그.

## 사전 조건

기존 코드에 테스트가 충분한가?

- ✅ 충분: 바로 진행
- ❌ 부족: **characterization test 먼저** — 현재 동작을 그대로 잠가두는 테스트를 추가하고 시작

## 절차

1. 안전망 확보 (테스트 통과 상태)
2. 작은 단계로 변경 (한 번에 한 가지)
3. 각 단계마다 `npm test` 통과 확인
4. 빨간불 나면 즉시 되돌림 — 끌고 가지 말 것
5. 단계가 누적되면 중간 커밋 (작게 자주)

## 리팩 신호

- 같은 코드 3번째 등장 → 추출
- 함수/컴포넌트가 한 화면을 넘어감 → 분해
- 이름이 의도를 못 표현 → 개명
- props가 7개 넘음 → 객체 그룹핑 또는 컴포넌트 분리

## 금지

- "이참에 같이"로 다른 변경 끼우기
- 테스트 없는 영역을 무방비로 건드리기
- 한 PR에 리팩 + 기능 동시

## 커밋

`♻️ [refactor] {대상} 구조 개선 — {핵심 변경 1줄}`
