# 워크플로우: 버그 수정

## 1. 재현

- 버그 정확한 입력/조건/결과 기록
- 가능한 한 작은 재현 케이스로 축소

## 2. RED (재현 테스트)

- **수정 전에** 실패하는 테스트를 먼저 작성
- 이 테스트가 버그를 그대로 표현해야 함
- 한 번 실행해서 실패 확인 (실패 사유가 버그와 일치하는지)

## 3. GREEN (최소 수정)

- 테스트만 통과시키는 가장 작은 변경
- 주변 정리는 다음 단계로 미룸

## 4. REFACTOR (선택)

- 같은 종류 버그가 또 날 위험이 있으면 구조 개선
- 단순 오타/누락은 리팩 생략 가능

## 5. 회귀 방지

- 이 테스트는 영구히 남는다 (지우지 말 것)
- 같은 영역 다른 시나리오 테스트도 추가 검토

## 커밋

`🐛 [fix] {증상} 수정 — {원인} 제거`

예: `🐛 [fix] 검색 결과 0건 시 무한 로딩 — empty 분기 누락`
